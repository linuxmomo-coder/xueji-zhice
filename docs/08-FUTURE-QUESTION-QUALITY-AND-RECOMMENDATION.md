# 学迹智评后续升级设计：题目勘误与相似题型推荐

> 状态：需求收集阶段，暂不进入当前 MVP 开发。
>
> 目标版本：V1.1/V1.2，具体排期以后续用户量、题库规模和数据质量决定。

## 1. 背景与目标

当前学习闭环依赖本地题库，因此题目质量和推荐准确性会直接影响错题本、掌握度和 AI 报告。后续升级需要增加两项能力：

1. 学生发现题目、答案或解析存在问题时，可以提交“勘误”。系统调用 AI 对问题进行独立复核，形成判断与修正建议；确认题目确实错误后，发布正确版本。
2. 系统不只按知识点推荐题目，还要归纳题型。当学生在某一种具体题型上持续出错时，优先推荐同类型、同能力要求、不同表述的相似题，形成专项训练。

## 2. 功能一：学生题目勘误机制

### 2.1 使用入口

学生可在以下页面点击“题目勘误”：

- 做题页面；
- 提交答案后的解析页面；
- 错题本详情页；
- 复测结果页。

### 2.2 勘误类型

学生提交时至少选择一类：

- 题干表述错误；
- 条件缺失或条件冲突；
- 选项重复、缺失或无唯一答案；
- 标准答案错误；
- 解析错误或解析与答案矛盾；
- 图片、公式、单位或符号显示错误；
- 知识点、年级、难度或教材映射错误；
- 其他问题。

可选填写：

- 学生认为正确的答案；
- 问题说明；
- 对应步骤或证据；
- 是否影响本次判分。

### 2.3 状态流程

```text
student_submitted
    ↓
ai_reviewing
    ├── likely_invalid       学生反馈大概率不成立
    ├── likely_valid         AI判断大概率存在错误
    └── uncertain            AI无法可靠判断
            ↓
admin_reviewing / expert_reviewing
    ├── rejected             反馈不成立
    ├── correction_draft     生成修订草案
    ├── corrected            新版本已发布
    └── retired              题目停用且不再发布
```

### 2.4 AI复核内容

AI不得只阅读原解析后复述结论，而应执行独立复核：

1. 独立求解题目；
2. 检查题干条件是否充分、是否矛盾；
3. 检查选择题是否存在唯一正确答案；
4. 比对标准答案、解析和独立求解结果；
5. 检查单位、公式、符号及数值计算；
6. 检查题型、知识点、年级和难度标签；
7. 输出问题类型、证据、置信度和修订建议。

AI输出必须为结构化结果，例如：

```json
{
  "verdict": "likely_valid",
  "confidence": 0.96,
  "error_types": ["answer_error", "explanation_conflict"],
  "independent_answer": {"value": "B"},
  "evidence": ["根据题干条件计算结果为B", "原解析第二步计算符号错误"],
  "correction_proposal": {
    "answer": {"value": "B"},
    "explanation": "修正后的解析"
  },
  "affects_scoring": true
}
```

### 2.5 发布控制原则

AI可以完成初审、生成修订草案和风险分级，但默认不直接覆盖生产题目。推荐采用以下安全规则：

- AI高置信度判断错误时，立即将题目标记为“疑似错误”，暂停继续推荐；
- 创建题目新版本，不覆盖旧版本；
- 由后台管理员或学科审核人确认后发布；
- 简单客观题后续可评估“双模型一致 + 规则校验 + 高置信度”的自动发布模式；
- 主观题、开放题、存在多种合理答案的题目必须人工审核。

### 2.6 修正后的连带处理

如果题目错误影响判分，系统必须执行数据修复：

1. 找出受影响的答题记录；
2. 按新答案重新判分；
3. 删除或修正因错误答案产生的错题记录；
4. 重新计算相关知识点掌握度；
5. 修正任务完成结果和奖励积分；
6. 标记受影响的 AI 报告需要重新生成；
7. 通知提交勘误的学生及受影响用户；
8. 保留修正前后版本、审核人和操作日志。

不得静默修改历史数据，所有重算必须可追溯。

### 2.7 防滥用与激励

- 同一学生对同一题只能保留一条未结束的勘误；
- 对短时间大量无效反馈进行限流；
- 学生不能看到其他学生身份和反馈内容；
- 有效勘误可授予“题库小卫士”等非现金成长徽章；
- 奖励不得仅依据提交数量，必须以最终确认有效为准。

## 3. 功能二：题型归纳与相似题推荐

### 3.1 题型不能只使用单一 question_type

`single_choice`、`fill_blank` 等只是呈现形式，无法说明学生究竟在哪类问题上出错。正式题型体系应至少包含以下维度：

| 维度 | 示例 |
|---|---|
| 展示题型 | single_choice、calculation、reading |
| 题型族 | 分数应用题、第三人称单数、联系上下文理解词义 |
| 题型模板 | 已知总量和份数求每份、主语为第三人称单数补全动词 |
| 核心知识点 | 分数的意义、一般现在时、词语理解 |
| 考查技能 | 建模、运算、信息提取、规则应用、迁移 |
| 认知层级 | remember、understand、application、analysis |
| 常见错误类型 | 条件识别错误、计算顺序错误、规则漏用、单位遗漏 |
| 难度特征 | 条件数量、计算步骤、干扰项强度、阅读量 |
| 表达特征 | 文字题、图表题、情境题、公式题 |

### 3.2 题目归类方式

题型标签来源可以包括：

1. 题库录入人员人工填写；
2. AI根据题干、答案和解析自动建议；
3. 后台审核人确认；
4. 学生真实答题数据反向修正题型标签。

AI自动分类结果必须保存置信度和模型版本，不能无记录覆盖人工标签。

### 3.3 学生个人题型错误画像

系统为每个学生建立题型错误画像，至少记录：

- 题型族和题型模板；
- 涉及知识点；
- 错误次数和最近错误时间；
- 连续错误次数；
- 原题、同类题和变式题通过情况；
- 是否使用提示；
- 平均答题时间；
- 主要错误类型；
- 当前掌握状态；
- 下一次推荐时间。

不能因为一次偶然错误就判断学生存在稳定弱项。建议满足以下任一条件后进入专项推荐：

- 同题型最近3题错误2题以上；
- 同题型连续错误2次；
- 正确率低于设定阈值且证据题数达到最低数量；
- 错题复测仍未通过；
- 答对但长期依赖提示或明显超时。

### 3.4 相似题推荐逻辑

推荐不是简单复制原题，而应同时满足：

1. 同核心知识点；
2. 同题型族或同题型模板；
3. 与学生已学教材和当前进度匹配；
4. 难度与当前能力相适应；
5. 排除近期重复题；
6. 答案、数字、情境或表述发生变化；
7. 保留可解释的推荐原因。

推荐优先级建议：

```text
原题理解与订正
    ↓
同模板低难度题
    ↓
同模板正常难度题
    ↓
同题型不同情境题
    ↓
跨表述变式题
    ↓
综合应用题
```

### 3.5 推荐分数示例

后续可采用可解释打分：

```text
recommendation_score =
    0.30 × 题型模板相似度
  + 0.25 × 知识点匹配度
  + 0.15 × 错误类型匹配度
  + 0.10 × 教材进度匹配度
  + 0.10 × 难度适配度
  + 0.05 × 复习时间紧迫度
  + 0.05 × 题目质量分
  - 重复与疲劳惩罚
```

结构化标签负责硬过滤，向量相似度只作为候选排序依据，不能单独决定推荐。

### 3.6 推荐停止与升级条件

- 连续通过2至3道同类型非重复题后，降低该题型推荐频率；
- 原题通过但同类题失败，不得标记稳定掌握；
- 同类题通过但变式题失败，继续推荐变式训练；
- 多轮仍未通过时，降低难度并回退到前置知识点；
- 长期稳定通过后进入间隔复习，而不是继续密集推荐。

### 3.7 推荐解释

学生端建议显示简短理由：

> 你最近在“已知总量和份数求每份”这类题上容易把除数和被除数写反，下面安排2道同类题巩固。

家长端显示更完整证据：

- 最近5题正确2题；
- 其中3题出现相同错误；
- 2次依赖提示；
- 当前推荐2道基础同类题和1道变式题。

## 4. 后续数据库扩展建议

### 4.1 question_error_reports 题目勘误单

建议字段：

- `id`
- `question_id`
- `question_version`
- `student_id`
- `answer_record_id nullable`
- `report_type`
- `description`
- `suggested_answer JSONB nullable`
- `affects_scoring_claimed`
- `status`
- `ai_verdict`
- `ai_confidence`
- `ai_evidence JSONB`
- `created_at/updated_at`

### 4.2 question_correction_reviews 勘误审核

- `error_report_id`
- `reviewer_user_id`
- `review_result`
- `review_note`
- `correction_patch JSONB`
- `published_question_version nullable`
- `reviewed_at`

### 4.3 question_versions 题目版本

每次修改保存完整快照：

- `question_id`
- `version`
- `snapshot JSONB`
- `change_reason`
- `source_error_report_id nullable`
- `created_by`
- `published_at`

### 4.4 answer_regrade_jobs 受影响答题重算任务

- `question_id`
- `from_version/to_version`
- `status`
- `affected_answer_count`
- `affected_student_count`
- `started_at/finished_at`
- `result_summary JSONB`

### 4.5 question_type_taxonomy 题型分类树

- `id`
- `subject_id`
- `parent_id nullable`
- `taxonomy_type`：family/template/skill/error_pattern
- `code`
- `name`
- `description`
- `status`

### 4.6 question_feature_tags 题目特征标签

- `question_id`
- `taxonomy_id`
- `source`：human/ai/system
- `confidence`
- `model_version nullable`
- `is_confirmed`

### 4.7 student_error_profiles 学生题型错误画像

- `student_id`
- `term_id`
- `taxonomy_id`
- `attempt_count`
- `wrong_count`
- `hinted_correct_count`
- `average_duration_seconds`
- `error_pattern JSONB`
- `status`
- `last_wrong_at`
- `next_recommend_at`

唯一约束建议：`student_id + term_id + taxonomy_id`。

### 4.8 recommendation_events 推荐记录

记录推荐原因、候选题、最终题目、模型/规则版本、是否完成及结果，用于后续评估推荐质量。

## 5. 对题库采集模板的后续影响

下一版题库 Excel 模板建议增加以下列：

- 题型族；
- 题型模板；
- 考查技能；
- 易错类型；
- 前置知识点；
- 表达形式；
- 题型标签审核状态；
- AI分类置信度；
- 题目版本；
- 是否允许自动勘误。

本轮暂不修改已生成模板，等该功能进入开发版本时再统一升级模板和导入程序。

## 6. 验收标准草案

### 6.1 勘误机制

1. 学生可针对具体题目和具体答题记录提交勘误；
2. AI返回结构化判断、证据、置信度和修订建议；
3. 疑似错误题在审核期间可暂停推荐；
4. 修订以新版本发布，不覆盖历史版本；
5. 影响判分时能够重算答题、错题、掌握度和报告；
6. 学生可看到勘误处理结果；
7. 全流程具备审计记录。

### 6.2 相似题型推荐

1. 题目可同时具备展示题型、题型族、题型模板和错误类型标签；
2. 系统能识别学生持续薄弱的具体题型；
3. 推荐题符合教材进度、知识点、题型和难度约束；
4. 推荐结果不是近期重复题；
5. 每次推荐都能说明原因；
6. 连续通过后自动降低推荐频率；
7. 推荐效果可以通过后续答题数据评估。

## 7. 开发优先级建议

建议分三期实施：

### V1.1-A

- 勘误入口；
- 勘误单和后台审核；
- AI独立复核；
- 疑似错误题暂停推荐；
- 题目版本管理。

### V1.1-B

- 受影响答题重算；
- 错题和掌握度修复；
- 用户通知与勘误徽章。

### V1.2

- 题型分类树；
- AI题型标签建议；
- 学生题型错误画像；
- 相似题和变式题个性化推荐；
- 推荐效果评估与自动调参。
