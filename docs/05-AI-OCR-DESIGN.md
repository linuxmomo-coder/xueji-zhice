# 学迹智评 AI 与 OCR 设计说明书 v0.2

## 1. 原则

AI/OCR是可替换概率型适配器，不能替代身份权限、确定性判分、数据确认和发布审核。

## 2. 当前状态

| 能力 | 状态 | 说明 |
|---|---|---|
| 文件私有上传、摘要和去重 | available | LocalPrivateStorage |
| 家长确认资料 | available | 人工确认基线 |
| PaddleOCR真实识别 | planned | 下一阶段异步任务 |
| 百炼/混元报告 | planned | 适配器和证据校验后接入 |
| 数学表达式判分 | available | 规则+安全AST+SymPy，不调用LLM |
| 题目勘误AI复核 | planned | AI初审，人工发布 |

## 3. OCR流程

```text
presigned/受控上传
→ MIME、大小、摘要和恶意文件检查
→ 私有存储
→ background_job: ocr_document
→ PaddleOCR
→ 文档类型结构化
→ 字段置信度
→ 家长确认/修订
→ 正式成绩或评语
→ 审计事件
```

关键数值低于阈值必须人工确认；学生上传永远不能自动成为正式记录。

## 4. AI报告流程

1. 权限校验并生成时间范围内证据快照；
2. 程序计算成绩趋势、正确率、提示率、复测和样本量；
3. 对外发送最小化、去标识数据；
4. 百炼生成结构化JSON；
5. JSON Schema和evidence_id校验；
6. 失败时切换混元；
7. 两者均失败时展示规则报告，不伪造AI结果。

结论等级：`fact/high_confidence/possible/insufficient_data`。

## 5. 数学判分边界

根号、分数、乘除、幂和圆周率由本地确定性解析器处理；国家智慧教育平台数学解题助手等外部服务只可用于后续题目勘误或解析对照，不能作为唯一判分来源。

## 6. 题目勘误

学生提交勘误后：

- AI独立解题并检查条件、答案和解析；
- 高风险题立即`suspended`停止推荐；
- AI输出置信度、证据和修订建议；
- 管理员/学科审核发布新版本；
- 影响判分时创建重判任务并保留前后记录。

## 7. 隐私与安全

- 不向外部模型发送姓名、邮箱、学校和联系方式；
- 原始图片默认不发给文本模型；
- 提示词和模型版本留痕；
- 防提示注入：用户文本作为数据字段，不拼入系统指令；
- 日志不记录完整学生原文和模型密钥。
