# 学迹智评软件需求规格说明书（SRS）

## 1. 文档目的

本说明书定义学迹智评 MVP 的业务范围、角色权限、功能需求、非功能需求、数据边界和验收标准，作为产品、设计、开发、测试和部署的共同基线。

## 2. 产品定位

学迹智评是一套以家长管理为核心、以本地题库为学习载体、以学校成绩与教师评语为外部评价依据、以学生真实答题与学习行为为内部证据的 AI 综合学习评价系统。

核心闭环：

1. 家长或学生拍照上传成绩、评语、学校评价或教材资料。
2. OCR 提取内容并形成结构化候选数据。
3. 家长确认后写入学生正式档案。
4. 系统按照年级、学期、教材、知识点和学习水平，从本地题库生成评测与练习。
5. 系统记录正确率、用时、提示、错题、复测和任务完成情况。
6. AI 生成学生版与家长版报告，并给出可执行的学习计划。

## 3. MVP 范围

### 3.1 包含

- 学生、家长、后台管理员三类角色。
- 家庭与学生绑定。
- 学年、学期、学制、行政年级和各科学习水平设置。
- 按科目设置教材出版社、版本、修订年份、册次和当前进度。
- 成绩、教师评语、学校评价、教材封面和目录拍照上传。
- OCR 结果人工确认、修改、驳回和历史留痕。
- 本地题库、教材映射、知识点、答案、解析、版权和审核状态管理。
- 诊断评测、日常练习、专项练习、错题重做、同类题、变式题和复测。
- 客观题自动判分。
- 知识点掌握度计算。
- 学生版和家长版 AI 报告。
- 家长任务推送、时长限制、打印和完成追踪。
- 后台用户、教材、题库、AI、OCR、日志与系统配置。

### 3.2 不包含

- 教师端、班级管理和教师批量操作。
- 整张试卷拆题 OCR。
- 纸质作业答案识别。
- 自动提取纸质错题。
- 教师红笔批改识别。
- 高风险主观题自动定分。
- 同学公开排名、社交互动和公开学习圈。

## 4. 用户角色

### 4.1 学生

- 查看今日任务、课程、知识卡、错题和成长记录。
- 完成本地题库练习、评测和复测。
- 查看学生友好版 AI 分析。
- 拍照上传成绩和评语资料；上传后进入待家长确认状态。
- 不能修改正式成绩和正式评语。

### 4.2 家长

- 创建和管理一个或多个学生档案。
- 设置年级、学期、教材、学习目标和每日时长。
- 上传、确认或修正成绩、评语和学校评价。
- 查看多维 AI 分析、成绩趋势和题库表现。
- 推送练习、评测、知识卡和学习计划。
- 管理数据导出、删除和隐私设置。

### 4.3 后台管理员

- 管理账号、权限、教材、知识点、题库、OCR 和 AI 配置。
- 审核题目、处理争议题、查看数据质量和运行日志。
- 默认不查看与运营无关的学生敏感原图和完整报告。

## 5. 功能需求

### FR-001 账户与家庭

- 家长通过手机号或邮箱注册。
- 家长可创建学生子账号，学生无需独立手机号。
- 一个家长可绑定多个学生；一个学生可绑定多个监护人。
- 必须有一名主监护人负责导出、删除和重要授权。
- 支持密码登录、刷新令牌、退出和设备会话管理。

### FR-002 学生基础档案

字段至少包括：昵称、出生年月、地区、学校（可选）、学制、当前学年、当前学期、行政年级、入学年份、各科学习水平、目标和每日可学习时间。

### FR-003 教材与课程设置

- 每个科目独立配置教材。
- 教材字段包括出版社、版本名、修订年份、课程标准版本、年级、册次。
- 支持地区推荐，但推荐结果必须由家长确认。
- 支持教材切换，历史记录不得覆盖。
- 目录层级：学段—年级—学期—科目—教材—单元—章节—课时—知识点。

### FR-004 学期归档与升年级

- 学期结束时冻结成绩、评语、掌握度和教材快照。
- 新学期重新确认年级与教材。
- 未掌握知识点可继承到新学期。
- 支持留级、跳级和各科学习水平不同于行政年级。

### FR-005 资料上传与 OCR

支持类型：成绩单、教师评语、学校评价、听写/小测成绩、教材封面、教材目录和学习进度资料。

上传流程：资料类型选择—拍照质量检测—OCR—结构化—置信度标记—人工确认—正式入档。

规则：

- 学生上传必须由家长确认。
- 家长上传也必须进入确认页面后保存。
- 低置信度字段标红。
- 保存原图、OCR 原文、结构化字段、修改前后值和确认人。
- 检测重复成绩和重复资料。

### FR-006 成绩管理

支持分数、满分、等级、班级平均分、班级排名、年级排名、考试类型、考试范围和数据来源。系统内部换算值不得覆盖学校原始评价制。

### FR-007 评语结构化

结构化维度至少包括：课堂专注、参与度、基础知识、理解能力、迁移应用、表达能力、作业完成、订正习惯、独立性、时间管理、主动性、合作表现、情绪信心和教师建议。

每个标签必须保存证据句、置信度、来源和家长确认状态。

### FR-008 本地题库

题目必须具备：题目 ID、学段、年级、学期、科目、教材映射、知识点、题型、难度、认知层级、预计用时、题干、选项、答案、解析、分步提示、常见错误、来源、版权、审核状态和版本。

题目主体与教材映射必须分离，一道题可映射多个教材版本。

### FR-009 评测与练习

支持初始诊断、章节前测、日常小测、专项评测、单元评测、错题复测和遗忘复测。

组题必须排除未学内容，并控制知识点、难度、重复题和预计时长。

### FR-010 判题与答题记录

- 客观题自动判分。
- 记录学生答案、用时、提示次数、提交次数、设备和中断情况。
- 主观题 MVP 只支持家长确认或规则型评分，不做高风险自动定分。

### FR-011 错题与复测

错题只来源于系统本地题库。状态包括：新错题、待学习、已看解析、待重做、重做失败、原题通过、同类题通过、变式题通过、稳定掌握、疑似遗忘。

仅做对原题不能直接标记为稳定掌握。

### FR-012 掌握度

综合最近正确率、题目难度、提示、用时、重复错误、同类题、变式题、复测、时间衰减和数据量计算。状态至少包括：未学习、数据不足、初步接触、需要补基础、正在学习、基本掌握、稳定掌握、可提高、疑似遗忘、需重新评测。

### FR-013 学生端激励

支持成长星、连续学习、知识点徽章、坚持徽章、独立完成徽章和错题逆袭徽章。奖励必须与具体学习行为关联，并设置防刷和每日上限。

### FR-014 家长任务推送

家长可从推荐计划中确认并推送任务，设置截止时间、每日时长上限、最晚学习时间、自动顺延和奖励。系统应避免总任务量超限。

### FR-015 AI 报告

学生版：最近进步、优势、小挑战、证据、任务、方法、鼓励、数据不足和更新时间。

家长版：执行摘要、数据范围、成绩趋势、评语汇总、题库表现、投入效率、多维分析、优势问题、可能原因、两周措施、沟通建议、风险和复评条件。

每个 AI 结论必须标记事实、较高可信判断、可能情况或数据不足。

### FR-016 打印

支持学生答题版、答案版、解析版、家长检查版、节约纸张版和大字版。纸面答题结果 MVP 通过家长手工录入或重新在线复测进入系统。

### FR-017 后台管理

包括用户、教材、课程标准、目录树、知识点、题库、题目映射、版权、审核、AI 模型、OCR 配置、报告模板、系统参数、运行状态、操作日志和数据质量监控。

## 6. 非功能需求

### NFR-001 性能

- 常规接口 P95 小于 800ms。
- 客观题提交反馈小于 1s。
- OCR 和 AI 报告采用异步任务。
- 题目检索 P95 小于 500ms。

### NFR-002 可用性

- 支持 PC、手机和平板 Web。
- 未提交答题在短暂断网后可恢复。
- 关键任务必须具备幂等性。

### NFR-003 安全

- 全站 HTTPS。
- 密码使用强哈希。
- 数据按家庭隔离。
- API 使用最小权限和角色授权。
- 原始图片使用私有存储和临时访问地址。
- API Key 不得进入前端、日志或 Git 仓库。

### NFR-004 隐私

- 未成年人数据最小化采集。
- 支持数据导出、纠正和删除。
- 真实数据不得默认用于公共模型训练。
- 调用外部 AI 前进行必要脱敏。

### NFR-005 可维护性

- 前后端分离。
- 模块化业务服务。
- 数据库迁移可追踪。
- 关键逻辑具备单元测试。
- AI 和 OCR 供应商通过适配器切换。

## 7. P0 验收标准

1. 家长可创建学生并设置年级、学期和分科教材。
2. 学生或家长可上传成绩/评语图片，系统返回可编辑 OCR 候选结果。
3. 学生上传资料必须经过家长确认后入档。
4. 后台可维护教材、知识点和题目，题目可映射教材章节。
5. 学生可完成客观题练习并形成答题记录和错题。
6. 错题可进行原题、同类题和变式题复测。
7. 系统可生成学生版和家长版结构化报告。
8. 家长可推送任务并查看完成状态。
9. 任何整张试卷图片不得进入自动拆题流程。
10. 敏感配置通过环境变量注入，仓库中无真实密钥。
