# 学迹智评软件需求规格说明书（SRS）v0.2.1

## 1. 文档状态

- 基线版本：0.2.1
- 适用范围：当前代码、测试、部署和后续迭代
- 产品形态：学生端、家长端、平台管理端 Web 应用
- 主要目标：先建立安全、真实、可审计的数据与学习闭环，再接入 OCR/AI

## 2. 产品定位

学迹智评通过家庭确认的学校资料、本地题库答题记录、错题复测和学习行为形成学生证据库，并在证据充分时提供学习评价、复习任务和家长建议。

系统必须区分：

- 事实：直接来自成绩、答题或确认记录；
- 较高可信判断：有足量且相互支持的证据；
- 可能情况：存在证据但不足以确定；
- 数据不足：不得生成确定结论。

## 3. 当前版本范围

### 3.1 已实现

- 家长邮箱注册和家庭自动创建。
- 登录时选择家长、学生或管理员身份，后端校验所选身份与账号角色一致。
- 访问令牌与刷新令牌轮换。
- 家庭成员、学生账号和学生档案关系。
- 学生数据按家庭或本人隔离。
- 学生档案创建、列表和详情。
- 题目稳定身份、题目版本、选项、作答字段、判分规则。
- 练习会话、题目版本快照、答题记录、归一化判分、错题记录。
- 选择题、标准化文本、数值容差和基础数学表达式等价判分。
- 成绩、评语等资料文件上传、摘要去重、私有保存和家长确认。
- 审计事件、统一错误响应和 request_id。
- 生产配置强校验、Alembic 迁移、CI 和 Docker 部署。
- 按登录角色展示独立导航和功能。
- 工作台指标、提示、练习科目和题量从数据库实时读取。

### 3.2 已设计但未完成

- PaddleOCR 真实识别和异步任务。
- 阿里云百炼主模型、腾讯混元备用模型。
- MinIO/COS 生产对象存储适配器。
- Excel 题库导入后台。
- 完整教材目录、知识点图谱和题型分类树。
- 同类题、变式题和间隔复习推荐。
- 学生题目勘误、AI复核和历史重判。
- 主观题人工复核工作台。

### 3.3 明确不包含

- 教师端和班级管理。
- 整张试卷自动拆题。
- 纸质作业答案识别。
- 自动提取纸质错题。
- 未经审核的 AI 生成题直接发布。
- 高风险主观题自动定分。
- 学生公开排名和社交功能。

## 4. 角色与权限

### 4.1 学生

- 查看本人工作台和本人档案摘要。
- 完成本地题库练习。
- 查看本人错题和复测状态。
- 上传资料，但不能确认正式成绩或评语。
- 不得访问其他学生和其他家庭数据。

### 4.2 家长

- 创建和管理本家庭学生档案。
- 上传、查看和确认绑定学生资料。
- 为绑定学生创建练习和查看错题。
- 查看家庭范围内的报告和任务。
- 不得访问其他家庭。

### 4.3 管理员

- 管理题库、教材、审核和系统配置。
- 查看脱敏运行指标。
- 默认不进入家庭日常资料和报告详情。
- 管理接口使用独立后端权限，不与前端视图混用。

## 5. 功能需求

### FR-001 认证与会话

- 家长通过邮箱和密码注册。
- 密码至少8位并使用强哈希保存。
- 登录返回短期访问令牌和可轮换刷新令牌。
- 刷新令牌在数据库保存摘要，可撤销且单次轮换。
- 退出撤销刷新会话。
- 登录请求必须携带所选角色；服务器校验账号实际角色与所选角色一致。
- 令牌中的角色仅由服务器签发，登录后不得在前端切换角色。
- 登录页不得预填或展示生产账号、密码或演示凭据。

### FR-002 家庭与成员

- 注册家长时创建家庭和主监护成员关系。
- 一个家庭可有多个监护人和学生账号。
- 家庭是家长数据访问边界。
- 跨家庭访问必须返回403。

### FR-003 学生档案

字段至少包括：昵称、出生日期、学制、行政年级、当前学期、地区、每日学习时长、状态、创建人和软删除时间。

- 家长只能创建本家庭学生。
- 学生账号只能关联一个学生档案。
- 列表接口必须分页。
- 重要变更必须写审计事件。

### FR-004 本地题库

- 题目身份与内容版本分离。
- 发布题目必须存在已审核、已发布版本。
- 题目版本保存题干、解析、难度、认知层级、预计用时、总分、内容摘要和审核信息。
- 选择题选项独立保存。
- 一道题可有多个作答字段。
- 每个作答字段可配置多条判分规则。
- 已发布版本不得静默覆盖。

### FR-005 判分规则

系统必须支持：

- 选择集合全等；
- 精确文本；
- NFKC 后的标准化文本；
- 大小写按题目配置；
- 全角半角等价；
- 数值绝对/相对误差；
- 根号、乘除号、幂和圆周率的基础数学表达式等价；
- 多答案有序或无序集合；
- 无法安全解析时判错或进入人工复核。

禁止使用 Python `eval` 直接计算学生输入。

### FR-006 练习与答题

- 家长、学生或管理员在权限范围内创建练习。
- 组题只选择 active 且具有 approved/published 版本的题目。
- 每个练习项保存题目ID、版本ID和题目快照。
- 下一题接口不得返回标准答案和完整解析。
- 同一练习项首次有效提交后不可重复覆盖。
- 答题记录保存原始答案、归一化答案、分数、正确性、提示次数、用时和判分明细。
- 前端可选科目必须来自学生当前年级的已审核发布题库。
- 前端可选题量不得超过当前筛选条件下的真实可用题数。

### FR-007 错题与复测

- 错答创建或更新学生错题。
- 状态至少包括：new、learning、retest_pending、retest_failed、original_passed、similar_passed、variant_passed、mastered、suspected_forgotten。
- 原题答对只能进入 original_passed，不能直接稳定掌握。
- 错题必须保存最近错误时间、错误次数、最新答题记录和下次复习时间。

### FR-008 学习资料上传

- 支持成绩、教师评语、学校评价、教材封面、教材目录和进度资料。
- 支持 PNG、JPEG、WebP 和 PDF 白名单。
- 文件大小受配置限制。
- 上传后计算 SHA-256，并按学生检测重复。
- 文件保存在项目控制的私有存储，不对公网直接暴露。
- 家长确认后才进入正式档案状态。
- 确认保存确认人、确认时间和前后审计数据。

### FR-009 Demo 隔离

- demo 路由只在 development/test 且 `ENABLE_DEMO=true` 时注册。
- 演示种子只在 `SEED_DEMO_DATA=true` 时创建。
- production 环境启用 demo 或演示种子必须启动失败。
- 生产界面不得展示演示账号、演示统计或用固定值模拟业务数据。

### FR-010 统一错误和追踪

错误响应统一为：

```json
{
  "error": {
    "code": "FAMILY_001",
    "message": "无权访问该学生数据",
    "request_id": "uuid",
    "details": {}
  }
}
```

每个响应返回 `X-Request-ID`，前端仅向普通用户显示可执行中文提示。

### FR-011 角色化工作台与实时数据

- 家长、学生、管理员登录后使用独立导航，不在登录后提供角色切换。
- `/dashboard`只依据访问令牌中的真实角色聚合数据，不接受前端传入角色。
- 已完成练习只统计状态为`completed`的会话。
- 待复习错题排除`mastered`记录。
- 待确认资料只统计`awaiting_confirmation`记录。
- 管理员题目数量只统计active且当前版本approved/published的题目。
- 数据为空时展示0或明确空状态，不使用固定数据填充。
- 响应必须带`generated_at`，界面显示本次数据更新时间。

## 6. 非功能需求

### NFR-001 安全

- 生产环境全站 HTTPS。
- 生产密钥不少于32位随机值。
- 禁止通配 CORS。
- 禁止生产 SQLite、demo 和自动建表。
- PostgreSQL、Redis 和上传目录不暴露公网。
- 不记录密码、令牌、API Key 和学生原始敏感内容。

### NFR-002 可维护性

- 模块化单体结构。
- Router 负责协议，Service 负责用例，Repository 负责查询，Adapter 负责外部系统。
- 数据库只通过 Alembic 迁移。
- 文档明确标记 available/planned，不使用 demo 接口冒充生产能力。

### NFR-003 性能

- 常规API P95目标小于800ms。
- 单次判题目标小于1秒。
- 列表最大 page_size 为100。
- OCR、AI、PDF等耗时操作后续进入异步任务。

### NFR-004 隐私

- 未成年人数据最小化采集。
- 文件私有保存，访问地址短期有效。
- 外部 AI 调用前脱敏。
- 真实数据不得默认用于公共模型训练。

### NFR-005 可用性和无障碍

- PC、手机和平板可操作。
- 所有表单有标签、错误提示和焦点样式。
- 不能仅用颜色表示状态。
- 移动端主要触控目标不小于44px。

## 7. v0.2.1 验收标准

1. 未登录访问学生接口返回401。
2. 家长A访问家长B学生返回403。
3. production 使用默认密钥、SQLite、通配 CORS 或 demo 时启动失败。
4. 应用启动不执行 `create_all`，Docker启动通过 Alembic迁移。
5. 开发环境可显式创建演示账号，生产环境不存在演示账号入口。
6. 已审核题目可以生成练习，未审核题目不能进入练习。
7. 每个练习项保存题目版本和快照。
8. 错答形成答题记录和错题；根式等价输入可正确判分。
9. 文件上传检测 MIME、大小、SHA-256 和重复。
10. 关键操作包含 request_id 和审计事件。
11. 登录所选角色与账号实际角色不一致时返回401。
12. 登录后导航只包含该角色允许的功能。
13. 工作台指标从数据库聚合，空库显示0或空状态。
14. 练习科目和题量来自该年级的实际已发布题库。
