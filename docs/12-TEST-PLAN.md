# 学迹智评测试计划 v0.2.1

## 1. 质量门禁

Pull Request必须通过：

- `pytest --cov=app --cov-fail-under=60`；
- Ruff静态检查；
- Bandit安全扫描；
- Alembic空库upgrade和downgrade演练；
- TypeScript检查和Vite构建；
- Docker Compose解析；
- 前后端镜像构建。

## 2. 已实现后端用例

| 范围 | 用例 |
|---|---|
| 健康/配置 | health；production弱密钥、demo、SQLite、CORS拒绝 |
| 认证 | 家长注册、角色登录、角色不匹配拒绝、refresh轮换、logout |
| 家庭隔离 | 家庭A不能读取家庭B学生；未登录不能访问 |
| 学生 | 创建、分页、详情与角色限制 |
| 工作台 | 指标从数据库实时聚合；空数据为0；返回生成时间 |
| 题库 | 只选择active且approved/published版本；按年级返回实际科目题量 |
| 练习 | 创建session、快照、next不泄露答案、首次提交 |
| 判分 | 选择集合、NFKC、大小写、数值容差、根式等价 |
| 错题 | 错答创建/累计wrong_question |
| 文件 | MIME/大小/摘要/重复/确认权限（需继续扩充） |

## 3. 测试数据

测试使用临时SQLite和显式fixture。仅测试环境可显式加载虚构seed；生产配置禁止seed。每个测试重建数据库，不能依赖测试执行顺序。

## 4. 前端测试计划

下一阶段增加：

- Vitest：登录角色选择、错误映射、token存储、角色导航、loading和空状态；
- Vitest：数据库返回0时不得替换为固定展示数据；
- Playwright路径A：家长注册→创建学生→上传资料→确认；
- Playwright路径B：学生选择身份登录→练习→错答→错题→复测；
- Playwright路径C：账号选择错误角色时登录被拒绝；
- 无障碍：键盘、focus-visible、ARIA、对比度和移动触控目标。

## 5. 安全测试

- 401/403矩阵和IDOR；
- JWT类型、过期、签名和撤销；
- 登录角色声明与账号角色不匹配；
- 文件路径穿越、伪MIME、超限、重复和SVG；
- 数学表达式拒绝属性、函数注入和超复杂表达式；
- 密钥扫描、依赖漏洞和镜像扫描；
- 日志敏感数据检查。

## 6. 实时数据界面测试

至少覆盖：

1. 新注册家庭没有学生时，学生数、练习数、错题数、资料数均为0；
2. 创建学生后，家庭学生档案数量立即变为1；
3. in_progress练习不计入“已完成练习”；
4. completed练习计入“已完成练习”；
5. mastered错题不计入“待复习错题”；
6. 非awaiting_confirmation资料不计入“待确认资料”；
7. 无已发布题目时练习按钮不可用并显示原因；
8. `/questions/subjects`只返回目标年级实际存在的已发布科目和题量；
9. 前端题量选择不超过接口返回题量且上限为10；
10. 管理员、家长、学生导航不包含其他角色专属入口。

## 7. 上线验收

1. 未登录无法读取学生数据；
2. 跨家庭访问全部403；
3. production弱配置启动失败；
4. production不存在demo路由和seed；
5. 空库能迁移到head；
6. 练习保存版本快照，题库更新不改历史；
7. 全角、根号和等价表达式正确判分；
8. 错答生成错题；
9. 文件不经Nginx公开；
10. 登录所选角色与账号角色不一致时拒绝；
11. 登录后界面只显示当前角色功能；
12. 工作台不包含固定演示统计，科目和题量来自已发布题库；
13. CI全绿且部署后health通过。
